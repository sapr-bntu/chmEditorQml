Действия при сканировании

Для сканирований пользователь может настроить действие и параметры исправления.

Табл.: Варианты действий

Тип обнаружения

Варианты действий

Макровирусы и Прочие вирусы

Можно настроить первое действие и второе действие, выполняемое в случае сбоя первого.

Возможные действия:

  • Исправить угрозу: Программа попытается удалить обнаруженный вирус из зараженного файла (только в качестве первого действия).

  • Поместить угрозу в карантин: Сразу после обнаружения вируса предпринимается попытка переместить зараженный файл в карантин на зараженном компьютере. Файл в карантине нельзя запустить, если только он не будет восстановлен в исходном расположении.

  • Удалить угрозу: Предпринимается попытка удалить зараженный файл. Это действие следует выполнять только в том случае, если файл можно восстановить из незараженной копии. Удаленный файл нельзя восстановить из Корзины.

    Если клиенту не удается удалить файл, то подробная информация о выполненном действии появляется в окне уведомления, а также заносится в журнал угроз клиента.

  • Оставить как есть (только занести в журнал): Программа запрещает доступ к файлу, показывает уведомление и заносит информацию о событии в журнал. Выберите этот вариант, если вы хотите управлять способом обработки вирусов вручную.

    Действие для угрозы можно выбрать в журнале угроз.

Угрозы безопасности

Действия по обработке угроз безопасности можно настроить следующим образом:

  • Настроить одинаковые действия для всех угроз безопасности.

  • Настроить одинаковые действия для всех угроз из определенной категории.

    Щелкните на угрозе и выберите Переопределить действия, настроенные для угроз.

  • Исключить некоторые относящиеся к определенной категории угрозы из числа тех, для которых применяются общие действия.

Можно настроить первое действие и второе действие, выполняемое в случае сбоя первого. Для угроз безопасности могут выполняться следующие действия:

  • Поместить угрозу в карантин: Сразу после обнаружения угрозы или завершения ее установки предпринимается попытка переместить зараженный файл в карантин на зараженном компьютере. Функция сканирования удаляет или устраняет все побочные эффекты угрозы. К побочным эффектам могут относиться добавленные разделы реестра, измененные значения реестра, записи, добавленные в файлы .ini и .bat либо лишние записи в файле hosts. Кроме того, это могут быть ошибки в системном драйвере модуля LSP или изменения rootkit. Файлы из карантина, содержащие угрозы, можно восстановить в системе в исходном виде. В некоторых случаях для завершения устранения угрозы требуется перезапустить компьютер.

  • Удалить угрозу: Предпринимается попытка удалить файлы, содержащие угрозу. Это действие следует выполнять только в том случае, если файл можно восстановить из незараженной копии. Удаленные файлы нельзя восстановить из Корзины.

    Будьте осторожны при выборе этого действия. После удаления угроз безопасности в работе некоторых приложений могут возникнуть ошибки.

    Если клиенту не удается удалить файлы, то подробная информация о выполненных действиях отображается в окне уведомления и сохраняется в журнале угроз.

  • Оставить как есть (только занести в журнал): Никакие действия по обработке угрозы не предпринимаются, а информация об угрозе заносится в журнал. Выберите этот параметр, если следует вручную управлять тем, каким образом клиент обрабатывает угрозы. Действие для угрозы можно выбрать в журнале угроз.

Примечание:

Иногда можно случайно установить приложение, содержащее угрозу безопасности, такую как программа-шпион или программа показа рекламы. Если по оценке Symantec блокировка угрозы не принесет вреда компьютеру, то по умолчанию угроза будет заблокирована. Если блокировка угрозы может привести к нестабильной работе системы, клиент дождется окончания установки приложения. Затем будет выполнено действие, настроенное для угрозы.

Примечание:

Другие - это категория программ, представляющих собой угрозу, но не входящих в другие категории угроз.
