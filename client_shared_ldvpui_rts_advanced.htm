Дополнительные параметры автоматической защиты

Пользователь может настроить дополнительные параметры автоматической защиты файловой системы.

Табл.: Дополнительные параметры автоматической защиты файловой системы

Параметр

Описание

При необходимости перезагрузки автоматической защиты

Указывает, когда необходимо перезагружать автоматическую защиту.

Предусмотрены следующие параметры:

  • Отложить до перезапуска компьютера: Останавливает и перезагружает автоматическую защиту при перезапуске системы.

  • Перезагрузить автоматическую защиту: Останавливает и перезагружает автоматическую защиту немедленно.

Сканирование файлов

Предусмотрены следующие параметры:

  • Сканировать при обращении к файлу или его изменении: Сканирование файлов при их записи, открытии, перемещении, копировании и запуске. Данный параметр обеспечивает более надежную защиту системы. Выбор этого значения может привести к снижению производительности, поскольку функция автоматической защиты проверяет файлы при выполнении любых операций с ними.

  • Сканирование при изменении файла: Сканирование файлов при их записи, изменении и копировании. Это значение позволяет немного повысить производительность.

  • Сканировать при резервном копировании файла: Сканирование файлов при обращении к ним в процессе резервного копирования. Это значение можно выбрать в случае, если файлы, резервную копию которых предполагается создать, не сканировались. Использование данного параметра может существенно замедлить операцию резервного копирования, поскольку функция автоматической защиты будет проверять каждый сохраняемый файл.

    Данный параметр применяется только к файлам, для которых создается резервная копия. Функция автоматической защиты сканирует файлы, восстанавливаемые из резервной копии, вне зависимости от значения этого параметра.

Прочие параметры

Предусмотрены следующие параметры:

  • Удалять только что созданные зараженные файлы в случае действия 'Оставить как есть (занести в журнал)': Этот параметр позволяет удалять создаваемые зараженные файлы тех типов, для которых в автоматической защите настроено действие "Оставить как есть". Функция автоматической защиты не будет удалять файлы, которые уже существуют.

  • Сохранять данные о времени: Выберите этот параметр, если сведения о последнем обращении к файлу не должны изменяться. Если выбрано это значение, программа не будет создавать резервные копии неизмененных файлов.

Кэш файлов

Предусмотрены следующие параметры:

  • Выключить кэш файлов: Кэширование файлов сокращает объем памяти, используемый функцией автоматической защиты, и помогает отслеживать неполадки. Функция автоматической защиты добавляет в индекс кэш-памяти 16-разрядную запись, которая сохраняется там до тех пор, пока файл не будет изменен.

    Можно запретить кэшировать файлы при устранении неполадок.

  • Применять стандартный размер кэша: Размер кэша файлов по умолчанию считается на основе шаблонов использования среднего файла.

  • Укажите нестандартный размер кэша (кол-во файлов): Вместо стандартного размера будет применяться заданное количество записей кэша файлов. Позволяет указать количество включаемых в кэш записей. Этот параметр можно настроить для файловых и веб-серверов, которым приходится кэшировать много файлов.

  • Повторно сканировать кэш после загрузки новых описаний: Повторно сканирует кэш файлов после загрузки новых описаний на клиентский компьютер. При отключении этого параметра кэш файлов не сохранится после выключения компьютера клиента. После перезапуска клиентского компьютера автоматическая защита повторно просканирует все файлы.

Трассировщик угроз

Предусмотрены следующие параметры:

  • Включить трассировщик угроз: Включает или выключает трассировщик угроз.

  • Получать IP-адрес исходного компьютера: Если выбран этот параметр, функция автоматической защиты сохраняет только имя NetBIOS компьютера. Также сообщается имя пользователя, который работал в системе.

    Эта функция поддерживается только в системах Windows XP.

  • Опрос сетевых сеансов каждые: Включает или выключает опрос сетевых сеансов.

Автоматическая активация

Указывает, должна ли функция автоматической защиты повторно включаться через указанное количество минут.

Предусмотрен следующий параметр:

  • Если автоматическая защита выключена, включать через: Включает автоматическую защиту через указанное количество минут, если она была выключена. Этот параметр полезен в тех случаях, когда необходимо временно отключить автоматическую защиту. Допустимы значения от 3 до 60 минут.

Параметры резервного копирования

Предусмотрен следующий параметр:

  • Создавать резервную копию файлов перед исправлением: В качестве меры предосторожности перед исправлением файла необходимо включить этот параметр. Зашифрованная версия исходного, зараженного вирусом файла будет скопирована в каталог карантина. Этой копией файла можно воспользоваться для восстановления исходного зараженного файла.

    Примечание:

    Не рекомендуется отменять выбор этого параметра, так как в этом случае перед исправлением зараженных файлов не будут создаваться резервные копии файлов.

    Этот параметр действует только для файлов, зараженных вирусами. Клиент всегда создает резервные копии помещаемых в карантин и исправляемых файлов, вне зависимости от значения этого параметра.

Дополнительные расширенные параметры

Предусмотрены следующие параметры команды:

  • Эвристика: Позволяет изменить уровень защиты, который обеспечивается технологией эвристического сканирования Bloodhound.

  • Дискеты: Позволяет настроить параметры сканирования дискет.

  • Сеть: Позволяет настроить параметры сканирования сети.
